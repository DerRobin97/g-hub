Spezifikation für den 1:1-Nachbau

Dieses Dokument beschreibt das komplette visuelle System, jede Seite und jede Animation der G-Hub-App (iPhone, iPad-Querformat & Web) so vollständig, dass die Oberfläche pixel- und bewegungsgenau rekonstruiert werden kann. Die Ladeanimation liegt als einsatzbereite Datei bei.

Kapitel 1

Architektur & Aufbau

G-Hub ist eine mobile-first Marketing-Management-App für ein Garten- & Forsttechnik-Fachgeschäft (Gerber). Sie wird in drei Gehäusen ausgeliefert, teilt sich aber einen einzigen Quellcode.

Geräte & Renderziele

ZielMaße (Designgröße)Besonderheit
iPhone402 × 874 pxBottom-Tab-Navigation, KI als Bottom-Sheet, schwebender KI-Button
iPad · Querformat1194 × 834 pxApp-Spalte (~70 %) + andockbares KI-Panel rechts (~30 %)
Webresponsiveigene Datei G-Hub - Web.html, gleiche Routing-Logik

Der gemeinsame Frame skaliert das fixe Geräte-Layout per transform: scale() mittig in den Viewport (Letterboxing auf #0a0b0d). Bedienelemente liegen außerhalb des skalierten Bereichs.

Technischer Stack

  • React 18.3.1 über UMD + Babel Standalone (Inline-JSX, kein Build-Schritt).
  • Reines CSS mit Custom-Properties (Design-Tokens) — keine Utility-Frameworks.
  • Vanilla-Web-Component für den Ladescreen (<gerber-hub-loader>), framework-unabhängig.
  • Zustand pro App in React-State; Darstellung (Theme/Akzent) & Geräte-Position in localStorage.

Kapitel 5

Ladeanimation (Splash)

Der Ladescreen ist eine eigenständige Web-Component <gerber-hub-loader> (Datei quelldateien/gerberhub-loader.js) — keine Abhängigkeiten, läuft in React, Vue, Angular und purem HTML. Unten läuft die echte Animation.

Hinweis: läuft mit min-visible auf Dauerschleife, damit man sie hier beobachten kann. In der App blendet sie nach ~3,4 s aus.

Aufbau

  • Glow: radialer, weich geblurter Kreis hinter dem Logo (Akzentfarbe, 14 % Deckkraft).
  • Logo-Lockup: Vektor-Schriftzug „Gerber“ (einzeln animierte Buchstaben .ltr + Kreis-Icon .ico) und rechtsbündig darunter das Wort „Hub“ in Baloo 2 800 mit schwarzer Kontur (-webkit-text-stroke), Farbe = Akzent (Orange #ee7203).
  • Ring-Spinner: unten zentriert, 38px, 4px Rand, oberer Bogen in Akzentfarbe, Dauerrotation.
  • Version: optional ganz unten als Mono-Text, 34 % Deckkraft.

Zeitlicher Ablauf (Tempo speed=0.85, Zeitskala --ghl-t = 1/speed)

Glowghl-glow · 1.1s × t · ease · fadet ein (Opazität 0→1)
Buchstaben „pop“ghl-pop · 0.5s × t · cubic-bezier(.2,1.55,.35,1) · von scale(.2)→1, gestaffelt: Delay 0.45s×t + index·0.11s×t
Kreis-Iconghl-ring · 0.6s × t · cubic-bezier(.2,1.7,.35,1) · von scale(0) rotate(-120°)→ scale(1) rotate(0)
„Hub“-Wortghl-hubpop · 0.5s × t · cubic-bezier(.2,1.6,.4,1) · von translateY(-.35em) scale(.7)→1, Delay 1.25s×t
Spinner fadet einghl-fade · 0.6s × t · Delay 1.5s×t, danach ghl-spin 0.8s linear ∞
Endzustand sichernKlasse .ghl-settled nach 2100×t + 300ms → alle Elemente fix sichtbar (Fallback)

Ausblenden & Reveal-Übergabe (Frame-Regie)

Im Geräte-Frame steuert mountSplash() den Ablauf: Mindestanzeige ~3,4 s, dann:

  1. window.__appReady = true und Event app:reveal wird gefeuert (startet die Screen-Reveals der App).
  2. Das Lockup skaliert auf scale(1.5) und fadet (0.6 s, cubic-bezier(.6,0,.28,1)).
  3. Die ganze Component skaliert auf 1.08, blurrt 4px und fadet (Transform-Origin 50% 42%), dann remove() nach 720 ms.
Theming des Splash: Hintergrund kommt aus dem App-Theme (splashBg() → Schwarz/Grau/Weiß), der Akzent ist immer das Marken-Orange #ee7203 (themen-unabhängig). Reduzierte Bewegung (prefers-reduced-motion) zeigt sofort den Endzustand.

API der Component

<!-- pures HTML -->
<script src="gerberhub-loader.js"></script>
<gerber-hub-loader bg="#101217" accent="#ee7203"
   speed="0.85" min-visible="1800" version="Version 2.6.0"></gerber-hub-loader>

// ausblenden, sobald App bereit:
document.querySelector('gerber-hub-loader').hide();   // Promise
el.replay();                                          // neu abspielen
// Event 'gerberhub:hidden' nach Fade · Attribut auto-hide → bei window.load
Kapitel 6

Bewegungssprache

Bewegung ist ruhig, federnd und zweckgebunden. Es gibt zwei Standard-Easings, die fast überall wiederkehren. Alle dekorativen Animationen respektieren prefers-reduced-motion.

EasingKurveCharakter / Einsatz
Soft-Outcubic-bezier(.22,1,.36,1)ruhiges Ankommen — Screens, Reveals, Charts, Balken
Springcubic-bezier(.34,1.56,.64,1)federnder Overshoot — FAB-Dock, Sheets, Nav-Collapse
Material-Standardcubic-bezier(.4,0,.2,1)Breiten-/Layout-Transitions (iPad-Spalte, Nav)

Globale Animationen

Screen-EintrittscreenIn · 0.42s · Soft-Out · translateY(8px)→0 (jeder Tab-/Routenwechsel)
Entrance-RevealrevFade · 0.5s · Soft-Out · Opazität 0→1, pro Element gestaffelt via --rd (z. B. 40ms, 130ms …). Gesteuert vom useReveal-Hook + IntersectionLogik beim Scroll.
Count-up ZahlenrAF-Tween 900ms, Ease 1−(1−p)³; CountUp-Komponente, mit Fallback auf Endwert (Hintergrund-Tabs)
Sheet öffnensheetUp · 0.4s · Soft-Out · translateY(100%)→0. Varianten: Suche federt (Spring), KI nutzt „Spark-Slide“ (s. u.)
Sheet schließenWAAPI 0.3s · cubic-bezier(.4,0,1,1) · → translateY(100%) scale(.985) + Scrim-Fade; zentrierte iPad-Sheets behalten translateX(-50%)
Scrimfade · 0.25s · rgba(0,0,0,.55) + backdrop-blur(3px)
Chat-NachrichtmsgIn · 0.32s · Soft-Out · translateY(8px)→0; Tipp-Indikator aiBlink 1.2s ∞ (3 Punkte versetzt)
KI-FAB PulsaiPulse · 2.8s ∞ · expandierender Box-Shadow-Ring in Akzentfarbe

Nav-Collapse beim Scrollen

Sobald scrollTop > 18px, klappt die Bottom-Nav ein (nav-collapsed): Labels verschwinden (max-height 14→0), Icons schrumpfen auf 19px, der zentrale FAB wird von 54→30px. Alle Übergänge 0.42s cubic-bezier(.4,0,.2,1). Beim Routenwechsel wird die Nav wieder ausgeklappt.

KI „Spark-Slide“ (nur Handy-Sheet)

Beim Öffnen des KI-Assistenten wird per WAAPI an der FAB-Position (unten rechts) ein Funke gezündet, dann gleitet das Sheet hoch:

// 1) Funken-Burst am FAB
spark.animate([
  {opacity:1, transform:'scale(.3)',  boxShadow:'0 0 0 0 var(--accent-glow)'},
  {opacity:0, transform:'scale(2.8)', boxShadow:'0 0 0 30px transparent'}
], {duration:460, easing:'cubic-bezier(.2,.7,.3,1)'});
// 2) Sheet slide-up mit Overshoot, 150ms später
el.animate([{opacity:0, transform:'translateY(82%) scale(.97)'},{opacity:1}],
  {duration:520, delay:150, easing:'cubic-bezier(.34,1.4,.4,1)'});

Kapitel 8

App-Shell & Navigation

Die Shell (main.jsx → App) besteht aus drei festen Zonen: Header, Scroll-Body, Bottom-Nav — plus schwebendem KI-Button und aktivem Sheet.

Header

Links optional ein reiner Zurück-Pfeil (.back-btn, schiebt bei Hover −2px). Mitte: Kicker (Mono, uppercase) + Titel (Display 22px). Der Startseiten-Titel ist tageszeitabhängig: „Guten Morgen/Tag/Abend, Robin“. Rechts: Such-Button und kombinierter Mitteilungs-Button mit zwei Badges — links (Akzent) = ungelesene News, rechts (rot) = Benachrichtigungen.

Bottom-Navigation (Liquid Glass)

Fünf Slots: Startseite · Projekte · [＋-FAB] · Analytics · Profil. Das .nav-wrap ist im Dark-/Grau-Modus echtes „Liquid Glass“:

background: radial-gradient(120% 100% at 16% -25%, rgba(255,255,255,.22), transparent 56%),
            linear-gradient(158deg, rgba(255,255,255,.16), rgba(255,255,255,.035) 42%, rgba(255,255,255,.085)),
            color-mix(in oklab, var(--surface) 30%, transparent);
backdrop-filter: blur(30px) saturate(1.9) brightness(1.05);
box-shadow: 0 22px 50px -20px rgba(0,0,0,.8), inset 0 1.5px 1px rgba(255,255,255,.5), …;

Im hellen Modus wird daraus eine solide weiße Karte (kein Blur) mit weichem Schatten. Aktiver Tab: Text --text, Icon in Akzentfarbe mit Glow-drop-shadow. Der zentrale FAB (54px, Akzent) öffnet das „Erstellen“-Sheet.

Routing-Modell

  • Tabs: home · projekte · analytics · profil.
  • Projekt-Stack: innerhalb von Projekte und Profil ein Push-/Pop-Stack (z. B. planer → planerPost, pmProject → pmTask). Ein Verlaufs-Stack sorgt dafür, dass „Zurück“ immer genau eine Ebene zurückführt.
  • Sheets: overlay-basiert, unabhängig vom Tab (Liste in Kap. 9).

Tweaks-Panel

Über die Toolbar zuschaltbar. Standardmäßig zwei Tweaks: Akzent (Farb-Swatches inkl. der vier Paletten) und Ecken (soft/standard/sharp). Defaults: accent:["#cdf24d","#161a07"], corners:"standard".

Kapitel 12

iPad-Querformat

Gleiche App, andere Raumaufteilung. Klasse .ipad-land auf dem Screen-Container.

  • Spaltenlayout: App nimmt links ~70 %, das KI-Panel schwebt rechts (~30 %). .app{right:30%}, eingeklappt right:0; Übergang 0.44s cubic-bezier(.4,0,.2,1).
  • KI-Panel: .ai-float rundum abgesetzt, eigener Rahmen + großer Schatten. Eingeklappt gleitet es nach rechts raus (translateX(130%) + Fade); eine vertikale Griff-Lasche „KI“ (Akzent) holt es zurück.
  • KPI-Raster 4-spaltig, Werte größer (Fokus-Wert 40px, Chips 21px).
  • Sheets zentriert (max 720px, translateX(-50%)) und steigen mittig mit sanfter Skalierung hoch (sheetUpPad) — kein seitlicher Sprung. Das Arbeitszeit-Sheet öffnet als echtes Vollbild (sheetUpFull), zieht die App auf volle Breite und blendet das KI-Panel aus.
  • Statusbar-Attrappe oben (9:41 + WLAN/Akku-Glyphen).
  • Darstellungs-Änderungen synchronisieren über das Event ghub:appearance alle Instanzen (iPhone + iPad-Dock); kurzes Aussetzen der Transitions (.no-anim) verhindert Farb-Artefakte beim Theme-Wechsel.
Dev-Tool: Der Geräte-Frame enthält eine schwebende, frei verschiebbare Dev-Schaltfläche (Position in localStorage) mit Aktionen: Geräte neu starten, Ladescreen abspielen, Hell/Dunkel, Akzent wechseln, Seite neu laden. Nur für die Vorschau, nicht Teil des Produkts.
